## 2. Background-job framework / wayback archival

**Status**: `ready`  
**Plan**: [`plans/wayback-archive-job.md`](plans/wayback-archive-job.md)

Introduce a lightweight convention for scheduled background jobs that produce content changes via PR. Wayback archival is the first concrete job: `dr archive --pending` walks `research/sources/` and writes `archived_url` into source frontmatter; a daily GitHub Actions workflow commits the diff and opens a PR.

Punted from v1 because: the interim in-pipeline `skip_wayback=False` default (commit `6409918`) already ensures analysts see archived copies of paywalled sources during synchronous runs. The background-job framework is an improvement to throughput and operator ergonomics, not a correctness fix. When this ships, flip `skip_wayback` back to `True` in `VerifyConfig` and delete the interim note from `plans/wayback-archive-job.md`.

See the plan for full spec: CLI shape, throttle/retry, cooldown cache, GitHub Actions workflow, and implementation steps.

## 5. Source pool expansion — Tier 1

**Status**: `in progress` — schema + shared infrastructure done; Paths 1–3 ready to start  
**Plan**: [`plans/source-pool-expansion-tier1.md`](plans/source-pool-expansion-tier1.md)  
**Companion plan**: [`plans/source-pool-expansion-tier1-search-backend.md`](plans/source-pool-expansion-tier1-search-backend.md)

Adds three never-paywalled, mostly-independent acquisition surfaces (Wayback gap-fill, academic APIs, SEC EDGAR) to the Researcher → Ingestor pathway, plus the shared infrastructure all paths depend on.

**Done 2026-05-08** (commits `a40a09f`, `9a26ba8`, `0e5b1ff`, `98d094c`, `eb200f2`, `11aef7e`, `ee63cae`, `cfc8900`):

- Schema: `kind: 'paper'`, per-URL `acquisition` trace on `auditSchema.sources_consulted[]`, optional 10-digit `sec_cik` on company entities; `SourceKind.PAPER` mirror.
- Shared infra: `pipeline/common/throttle.py` (per-host async token bucket), `pipeline/common/canonical_url.py` (canonical-equivalence dedup, wired into `decomposed.py:execute_searches`), `_write_audit_sidecar` plumbing for per-URL `acquisition` and runtime `tool_outcomes`, `StepError.error_type` vocab docstring + drift smoke test, `VerifyConfig.research_origins: list[str] = ['brave']`.
- Observability: `dr stats --format text|json` (read-only) reporting wayback recovery rate, per-origin acquisition counts, verification_level distribution.

**Remaining**:

- Path 1: Memento secondary fallback for Wayback gap-fill + per-URL acquisition writes (~1 day).
- Path 2: arXiv + Semantic Scholar + OpenAlex tools, deterministic + classifier-driven affiliation override, `source-quality.md` amendment (~4–6 days).
- Path 3: SEC EDGAR tool, subject-relevance classifier, filer-vs-subject classification override, `source-quality.md` amendment (~5–7 days).
- Companion plan: search-backend swap (Tavily) behind `RESEARCH_SEARCH_BACKEND` (~4 days).

Paths are parallelisable; each is gated by inclusion in `VerifyConfig.research_origins`.
