## 2. Background-job framework / wayback archival

**Status**: `ready`  
**Plan**: [`plans/wayback-archive-job.md`](plans/wayback-archive-job.md)

Introduce a lightweight convention for scheduled background jobs that produce content changes via PR. Wayback archival is the first concrete job: `dr archive --pending` walks `research/sources/` and writes `archived_url` into source frontmatter; a daily GitHub Actions workflow commits the diff and opens a PR.

Punted from v1 because: the interim in-pipeline `skip_wayback=False` default (commit `6409918`) already ensures analysts see archived copies of paywalled sources during synchronous runs. The background-job framework is an improvement to throughput and operator ergonomics, not a correctness fix. When this ships, flip `skip_wayback` back to `True` in `VerifyConfig` and delete the interim note from `plans/wayback-archive-job.md`.

See the plan for full spec: CLI shape, throttle/retry, cooldown cache, GitHub Actions workflow, and implementation steps.
