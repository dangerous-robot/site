## 8. Semantic versioning

**Status**: `decision needed`

**Proposal**: `package.json` version field + git tags. `v0.1.0` is the first public release tag. DNS resolution is explicitly not part of the tag criteria — the tag records content/code readiness, not domain availability.

**Version semantics for a content site**:
- **Major**: breaking change to verdict enum, criterion definitions, or entity URL structure
- **Minor**: new entity type, new criterion, new capability shipped (e.g., audit trail, public feedback)
- **Patch**: content corrections, verdict updates, bug fixes, source additions

**Decision needed**: Display version in site footer? (Low effort: `import pkg from 'package.json'; {pkg.version}` in `Base.astro`.)
