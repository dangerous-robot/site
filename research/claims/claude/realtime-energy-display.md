The sources demonstrate that Claude provides real-time usage and cost monitoring through the Usage and Cost API and integrations like Grafana Cloud (Source 4), and supports telemetry export via OpenTelemetry for tracking usage and costs (Source 2). However, none of the sources indicate that Claude displays real-time *energy usage* specifically. Source 4 describes real-time visibility into 'usage, token consumption, and costs,' while Source 2 enables tracking of 'usage, costs, and tool activity'—but neither mentions energy consumption metrics. Source 3 is an independent third-party estimate of Claude's energy consumption, not an official display feature. The available monitoring tools track usage and costs, not energy.
